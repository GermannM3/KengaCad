# Руководство для разработчиков KengaCAD

## Обзор проекта

KengaCAD - это CAD-программа, максимально похожая на AutoCAD, предназначенная для настройки траекторий роботов нанесения мастики на кузова автомобилей с использованием движка Kenga через WebSocket API.

## Структура проекта

### Основные компоненты
- `main.py` - точка входа в приложение
- `kengacad_app.py` - основной класс приложения
- `ui/` - пользовательский интерфейс
- `engine/` - взаимодействие с движком Kenga
- `robot/` - управление роботами
- `cad/` - CAD функции
- `installers/` - файлы установщиков
- `build_scripts/` - скрипты сборки
- `examples/` - примеры использования API

## Продолжение разработки

### 1. Добавление новых команд в интерфейс

Для добавления новой команды:

1. Добавьте команду в `_parse_command` метод в `ui/main_window.py`:
```python
elif command == "NEW_COMMAND":
    self.statusBar().showMessage("Выполнение новой команды...")
    # Логика команды
```

2. Добавьте команду в автодополнение в `ui/command_line.py`

3. Если команда взаимодействует с движком, добавьте соответствующий метод в `engine/websocket_client.py`

### 2. Расширение API движка

Для добавления новой команды в WebSocket API:

1. Добавьте новую команду в `engine/websocket_client.py`:
```python
async def new_command(self, param1, param2):
    """Описание новой команды"""
    data = {
        "param1": param1,
        "param2": param2
    }
    return await self.send_command("new_command", data)
```

2. Реализуйте обработку команды в движке Kenga (в соответствующем модуле)

3. Обновите документацию в `API_DOCS.md`

### 3. Улучшение интерфейса

#### Ribbon-панель
- Файл: `ui/ribbon_bar.py`
- Добавление новых кнопок: используйте методы `addLargeButton` и `addSmallButton`
- Добавление новых панелей: используйте `addPanel` метод

#### Командная строка
- Файл: `ui/command_line.py`
- Добавление автодополнения: обновите список команд в `setup_completer`

#### 3D визуализация
- Файл: `ui/drawing_area.py`
- Для интеграции с движком Kenga используйте WebSocket API

### 4. Управление роботами

#### Добавление новых типов роботов
1. Создайте новый файл в `robot/` с определением робота
2. Добавьте поддержку в `robot/model.py`
3. Обновите конфигурационные файлы в `config/`

#### Улучшение кинематики
- Файл: `robot/model.py`
- Реализуйте прямую и обратную кинематику
- Добавьте проверки ограничений суставов

### 5. CAD функции

#### Траектории
- Файл: `cad/trajectory.py`
- Добавьте новые методы для работы с траекториями
- Реализуйте алгоритмы интерполяции и сглаживания

#### Импорт/экспорт
- Файл: `cad/import_export.py`
- Добавьте поддержку новых форматов
- Реализуйте конвертацию между форматами

### 6. Установщики

#### Windows (Inno Setup)
- Файл: `installers/kengacad_installer_fixed.iss`
- Для обновления: измените версию и добавьте новые файлы
- Запустите через ISCC для компиляции

#### Linux (DEB/RPM)
- Обновите соответствующие файлы в `installers/deb_package/` или `installers/rpm_package/`
- Обновите зависимости и структуру пакета

#### Arch Linux (AUR)
- Обновите `installers/arch_package/PKGBUILD`
- Обновите версию и зависимости

## Внесение изменений

### Git workflow
1. Создайте новую ветку: `git checkout -b feature/feature-name`
2. Внесите изменения
3. Протестируйте изменения
4. Создайте коммит: `git commit -m "Add feature description"`
5. Отправьте изменения: `git push origin feature/feature-name`
6. Создайте Pull Request

### Требования к коду
- Следуйте PEP 8
- Добавляйте docstrings ко всем функциям
- Используйте типизацию (typing module)
- Пишите понятные имена переменных и функций
