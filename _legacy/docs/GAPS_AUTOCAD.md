Документ фиксирует, чего не хватает KengaCAD для замены AutoCAD в типичных сценариях. Основа: `AUTOCAD_AUDIT.md`, `RELEASE_ROADMAP.md`, `COMMAND_REFERENCE.md`.

## 2. Чего не хватает для «полной» замены

### 2.1 Команды рисования и редактирования

| AutoCAD | KengaCAD | Приоритет |
|---------|----------|-----------|
| POLYGON | нет | Средний |
| DIVIDE | разбить объект на сегменты | Низкий |
| MEASURE | разбить по заданной длине | Низкий |
| REVCLOUD | облако ревизий | Низкий |
| ALIGN | выравнивание объектов | Средний |
| LENGTHEN | удлинение/укорачивание | Низкий |
| TRIM/EXTEND | есть, но UX проще в AutoCAD (обрезка к нескольким) | Средний |
| OOPS | восстановить последний ERASE | Низкий |

### 2.2 Аннотации и размеры

| AutoCAD | KengaCAD | Приоритет |
|---------|----------|-----------|
| MTEXT | многострочный текст | Средний |
| DIMANGULAR | угловой размер | Средний |
| DIMORDINATE | координатный размер | Низкий |
| DIMSTYLE | стили размеров | Высокий |
| LEADER / QLEADER | выноски | Средний |
| TABLE | таблицы | Низкий |

### 2.3 Слои и настройки

| AutoCAD | KengaCAD | Примечание |
|---------|----------|------------|
| LAYER фильтры | нет | Сложные чертежи |
| LAYER состояния | нет | Сохранение наборов |
| PROPERTIES | панель свойств | Правка толщины, цвета и т.д. |
| LTSCALE | масштаб типов линий | Глобальный |

### 2.4 Объекты и примитивы

| AutoCAD | KengaCAD | Примечание |
|---------|----------|------------|
| Регионы (REGION) | нет | Булевы операции над 2D |
| Мультилинии (MLINE) | нет | Специфика AutoCAD |
| Штриховка по границе | HATCH по выбору | Есть, но без UI выбора области |
| Стили текста | один шрифт | Шрифты, высота, стили |

### 2.5 Вид и навигация

| AutoCAD | KengaCAD | Примечание |
|---------|----------|------------|
| ZOOM Window | нет (масштаб к выделению есть) | Рамка для зумирования |
| ZOOM Previous | нет | Возврат к предыдущему виду |
| Named Views | нет | Сохранённые виды |
| Viewports (Paper Space) | нет | Листы, виды на листе |

### 2.6 Форматы и обмен

| AutoCAD | KengaCAD | Примечание |
|---------|----------|------------|
| DWG напрямую | через ODA | Нужен внешний конвертер |
| DWF/DWFx | нет | Просмотр/печать |
| PDF импорт | нет | Только экспорт |
| STEP/IGES | нет | 3D-обмен |

### 2.7 Печать и публикация

| AutoCAD | KengaCAD | Примечание |
|---------|----------|------------|
| Печать с листами | базовая | Масштаб, рамка |
| Пакетная печать | нет | Много листов |
| Стили печати (CTB) | нет | Цвет → толщина линий |

### 2.8 3D (если цель — 3D-CAD)

| AutoCAD | KengaCAD | Примечание |
|---------|----------|------------|
| EXTRUDE, REVOLVE | в движке | KengaCAD — 2D + робот |
| 3D-навигация | в движке | Окно 3D отдельно |
| Твёрдотельное моделирование | нет | Inventor/Fusion |

## 3. UX и интерфейс

| Элемент | Состояние |
|---------|-----------|
| Командная строка | Есть, Tab-дополнение |
| Панель свойств | Нет — редактирование через диалоги |
| Палитра слоёв | Панель Layers |
| Динамический ввод (ввод у курсора) | Нет |
| Гриппы (ручки) при выделении | Нет — только команды MOVE, ROTATE и т.д. |
| Правка «на месте» | Нет |

## 5. Итог по приоритетам

**Высокий (для восприятия как «почти AutoCAD»):**
- Стили размеров (DIMSTYLE)
- Панель свойств объектов
- Гриппы при выделении (перетаскивание, поворот)

**Средний (удобство):**
- POLYGON, ALIGN
- MTEXT, DIMANGULAR, LEADER
- ZOOM Window, ZOOM Previous
- Улучшенный TRIM/EXTEND (обрезка к нескольким)

**Низкий (специфика):**
- DIVIDE, MEASURE, LENGTHEN, OOPS
- DIMORDINATE, TABLE
- REGION, MLINE
- DWF, STEP, IGES

---

## 6. Профиль «CAD для роботов»

Для задачи **траекторий роботов нанесения** (основной профиль KengaCAD):

- Базовые команды рисования и редактирования — **достаточно**
- Робот, траектории, симуляция, диспенсинг — **есть**
- DXF/DWG — **есть**
- Не хватает в первую очередь: полировка UX (гриппы, свойства), печать.

Полная замена AutoCAD для универсального 2D-CAD — это отдельный большой объём работ; для роботистов текущий уровень уже пригоден.
