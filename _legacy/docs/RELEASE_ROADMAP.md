# Дорожная карта релиза KengaCAD

## Цель

Сделать KengaCAD полноценным аналогом AutoCAD и RoboCAD, готовым к демонстрации роботистам на заводе. Задача: взять EXE, показать, чтобы они купили и заменили RoboCAD.

---

## Текущее состояние (что уже есть)

### Интерфейс
- Ribbon в стиле AutoCAD (вкладки: Главная, Вставка, Аннотации, Вид, Симуляция, Робот)
- Командная строка с автодополнением (Tab)
- 2D область рисования (линии, окружности, дуги, точки, полилинии, текст)
- Snap: концы (E), середины (M), пересечения (I), центры (C)
- Панели: Objects, Layers, Snap Types
- Орто (F8), полярный (F10)

### CAD-команды (реализованы)
- `LINE x1 y1 [z1] x2 y2 [z2]`
- `CIRCLE x y [z] r`
- `ARC`, `POINT`, `POLYLINE`, `TEXT`
- DXF: импорт/экспорт (Open/Save)

### Робототехника
- `LOAD_ROBOT` — загрузка модели (glTF/GLB)
- `SET_JOINT` — установка угла сустава
- `TRAJECTORY` — создание траектории по точкам
- `SIMULATE` — симуляция
- `START_DISPENSING` / `STOP_DISPENSING`
- `CLEAR_SCENE`

### Инфраструктура
- Движок Kenga встроен в `engine_bin/kenga.exe`
- Автозапуск движка при старте
- WebSocket API для связи с 3D-движком
- 3D-визуализация — отдельное окно движка (Ebiten, 1280×720)

## Критичные пробелы для демо на заводе

### 1. Первый запуск «из коробки»

| Проблема | Решение |
|----------|---------|
| Движок запускается в фоне, окно 3D может не появиться | Проверить, что `CREATE_NO_WINDOW` не мешает GUI-окну Ebiten |
| Нет явной индикации «подключено к движку» | Добавить индикатор в статус-бар: зелёный = OK, красный = нет связи |
| При ошибке подключения — непонятно что делать | Диалог «Движок не запустился. Запустить вручную?» с инструкцией |

### 2. Демо-сценарий (обязательный flow)

Роботист должен за 5 минут увидеть:

1. **Запуск** → KengaCAD открывается, через 2–3 сек статус «Connected»
2. **2D чертёж** → `LINE 0 0 100 0` → `LINE 100 0 100 100` → линия видна
3. **Сохранение** → Файл → Сохранить → `test.dxf` → «Saved»
4. **Открытие** → Файл → Открыть → `test.dxf` → чертёж загружен
5. **Робот** → `LOAD_ROBOT` + путь к glTF → робот появляется в окне 3D
6. **Симуляция** → `SIMULATE` → робот двигается в 3D-окне

**Проверить:** есть ли тестовая модель робота в `assets/` или `samples/`? Если нет — добавить `robot.gltf` для демо.

### 3. Визуальная полировка

| Элемент | Статус | Действие |
|---------|--------|----------|
| `assets/logo.png` | Текстовый placeholder | Конвертировать `logo.svg` → PNG 256×256 |
| `assets/logo.ico` | Нет | Сгенерировать из PNG для иконки окна |
| Окно «О программе» | Есть | Добавить версию, ссылку на сайт/поддержку |

### 4. Команды из документации, но не реализованы

В `COMMAND_REFERENCE.md` указаны, в `_parse_command` отсутствуют:

- `MOVE`, `COPY`, `ROTATE`, `SCALE`, `MIRROR`, `ARRAY`
- `TRIM`, `EXTEND`, `BREAK`, `JOIN`, `OFFSET`
- `RECTANGLE`, `POLYGON`
- `ZOOM_EXTENTS`, `ZOOM_WINDOW`, `VIEW_TOP`, `VIEW_FRONT`, `VIEW_LEFT`, `VIEW_ISO`
- `UNDO`, `REDO` — заглушки

**Для демо достаточно:** `ZOOM_EXTENTS` (показать всё) и `UNDO` (отменить последнее). Остальное — после релиза.

### 5. Устойчивость

- При падении движка — переподключение или понятное сообщение
- При закрытии KengaCAD — корректное завершение движка (уже есть `_stop_engine`)

## Использование context7

При реализации новых фич можно использовать context7 для актуальной документации:

- **PyQt6** — виджеты, сигналы, стили
- **ezdxf** — импорт/экспорт DXF, сущности, слои
- **websockets** — переподключение, таймауты

Пример: при добавлении `ZOOM_EXTENTS` — проверить API `QGraphicsView.fitInView` через context7.
