# Агент-роботист для KengaCAD

Агент запускает KengaCAD, получает цель от пользователя и выполняет действия в приложении (кнопки ленты, командная строка), управляемый бесплатной нейросетью с Hugging Face (Inference API, без отдельного платного API-ключа).

## Требования

- Windows (pywinauto)
- Python 3.9+
- KengaCAD собран и доступен по пути (см. ниже)
- Токен Hugging Face (бесплатный) — только в переменной окружения

## Установка

```bash
cd agent
pip install -r requirements.txt
```

Токен Hugging Face не храните в коде. Задайте переменную окружения:

- **PowerShell:** `$env:HUGGINGFACE_TOKEN = "hf_..."`
- **CMD:** `set HUGGINGFACE_TOKEN=hf_...`
- Или создайте файл `.env` в папке `agent` (см. `.env.example`) и используйте `python-dotenv` в скрипте при необходимости.

Токен можно создать здесь: https://huggingface.co/settings/tokens (достаточно прав Read + Inference).

## Путь к KengaCAD

Агент ищет `KengaCAD.exe` в таком порядке:

1. `KengaCAD\publish\KengaCAD.exe` (результат publish)
2. `KengaCAD\bin\Release\net8.0-windows\win-x64\KengaCAD.exe` (сборка из репозитория)

Либо задайте путь в `config.py` (переменная или `get_kengacad_exe()`).

## Запуск

**Из корня репозитория:** `.\run_agent.ps1 "ваша цель"` (в PowerShell из папки `D:\KengaCAD`).  
Если KengaCAD уже открыт, агент подключится к нему и не будет запускать второе окно. Между действиями пауза 1,5 с — удобно следить за работой.

1. **Только запуск KengaCAD и ручные команды:**

```bash
python kengacad_driver.py start
python kengacad_driver.py click "Линия"
python kengacad_driver.py cmd "LINE"
```

2. **Агент по цели (нейросеть решает, что нажимать и что вводить):**

```bash
set HUGGINGFACE_TOKEN=hf_ваш_токен
python agent.py Нарисовать линию и круг и запустить симуляцию
```

Сначала агент запустит KengaCAD (если ещё не запущен), затем отправит цель модели на Hugging Face, получит список действий (CLICK/CMD/DONE) и выполнит их по очереди.

3. **Режим dry-run (без реальных кликов):**

```bash
python agent.py --dry-run Нарисовать прямоугольник
```

## Модель

По умолчанию в `config.py` указана модель `HuggingFaceH4/zephyr-7b-beta`. Она доступна в бесплатном Inference API при наличии токена. Если модель недоступна или требует соглашения, замените в `config.HF_MODEL` на другую (например, `microsoft/DialoGPT-medium` или любую из списка чат-моделей на Hugging Face).

## Структура

- `config.py` — путь к exe, токен из env, выбор модели
- `kengacad_driver.py` — запуск KengaCAD, клики по кнопкам, ввод в командную строку
- `hf_client.py` — запрос к Hugging Face Chat Completion API, разбор ответа на действия
- `agent.py` — цикл: цель → модель → разбор CLICK/CMD/DONE → выполнение в KengaCAD

Токен в коде не хранится и не коммитится; используется только переменная окружения `HUGGINGFACE_TOKEN`.

## Сценарии и как правильно работать

**Сценарий 1: Траектория полилинией (антигравий, кузов, пороги)**  
1. Запустите агента из корня: `.\run_agent.ps1 trajectory` или `.\run_agent.ps1 "траектория антигравий"`.  
2. Агент запустит KengaCAD, нажмёт «Полилиния» и введёт POLYLINE в командную строку.  
3. **Вы вручную**: кликайте по полю чертежа, ставя точки контура (пороги, контур кузова). Завершите полилинию: **Enter** или **двойной клик**.  
4. Для симуляции: откройте вкладку **Робот** и нажмите **Старт** — либо запустите агента снова: `.\run_agent.ps1 "start simulation"` (агент переключит вкладку Робот и нажмёт Старт).

**Сценарий 2: Только симуляция**  
Если траектория уже нарисована (линии, круги, полилиния):  
`.\run_agent.ps1 "start simulation"` — агент откроет вкладку «Робот» и нажмёт «Старт».

**Сценарий 3: Рисование линии/круга**  
В командной строке KengaCAD введите `LINE` или `CIRCLE` и рисуйте по полю; либо нажмите кнопку «Линия»/«Круг» на вкладке «Главная».

Без токена Hugging Face агент использует встроенные сценарии (полилиния + POLYLINE, при «start simulation» — вкладка Робот и Старт).
