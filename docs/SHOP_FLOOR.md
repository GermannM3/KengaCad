# Работа с ноутбука в цехе (без «роботического» жаргона)

На автозаводе (типа кузовного/герметика) обычно так:

1. Ноутбук подключают к **заводской Wi‑Fi или Ethernet** той же сети, где стоит шкаф робота (KUKA / Fanuc / ABB…).
2. В KengaCAD слева блок **«Цех — связь с роботом»**: вписывают **IP** с шильдика контроллера.
3. **Проверить** — слышен ли контроллер по сети (порт FTP 21 или UR 29999).
4. Справа собирают программу:
   - точки TCP (куда ехать),
   - **MoveL** между точками,
   - **Впрыск ВКЛ / ВЫКЛ** — включить/выключить пушку (DO3) на нужном участке шва.
5. **Экспорт+** — файл программы с движениями и сигналами впрыска.
6. **Залить FTP** — файл уходит на контроллер. Дальше пуск обычно с пульта/системы безопасности цеха.

## Что не делает программа «сама»

- Не обходит ограждение и не едет без человека у пульта.
- Номера выходов DO на шкафу могут отличаться (DO3 у вас может быть другим — переименуйте в таблице I/O или в коде после экспорта).
- OPC UA — связь с **PLC ячейки** (деталь на месте, безопасность), это отдельно от FTP на робот.

## Типичные бренды в кузове

| Бренд | Как заливаем программу | Впрыск в файле |
|-------|------------------------|----------------|
| KUKA | FTP → `/R1/Program` | `$OUT[3]=TRUE/FALSE` |
| ABB | FTP | `SetDO do3, 1/0` |
| Fanuc | FTP | `DO[3]=ON/OFF` |
| UR | Dashboard / скрипт | `set_digital_out(...)` |
