# Работа в цехе (Windows и Linux / Astra / Ред ОС)

## Реальность автозавода

На кузове обычно:

1. Ноутбук технолога в **заводской сети** (часто уже Linux: Astra, Ред ОС).
2. Связь с шкафом робота по **IP** (Ethernet).
3. Программа: траектория + когда включить пушку (**впрыск** / герметик / сварка).
4. Файл заливают на контроллер (**FTP** или через инженерный софт бренда).
5. Пуск — с пульта, не «из облака».

## Где что открывать

| Машина | Клиент | Блок «Цех» |
|--------|--------|------------|
| Windows-ноут | Setup / ZIP `KengaCAD.exe` | Слева «Цех — связь с роботом» |
| Astra / Ред ОС / Ubuntu | `*_linux-x64_portable.tar.gz` → `./run.sh` | Док **«Цех (Linux)»** |
| Android | APK | Вкладка «Связь» (FTP; CAD упрощён) |

Инструкция под отечественный Linux: [LINUX.md](LINUX.md).

## Пошагово (одинаково по смыслу)

1. Одна LAN с роботом.
2. IP с шильдика / от интегратора → **Проверить**.
3. Точки / полилиния.
4. **Впрыск ВКЛ / ВЫКЛ** (DO3 в файле — сверьте с электрикой ячейки).
5. **Экспорт+** → **Залить FTP**.
6. Пуск на роботе.

## Важно

- Программа **не едет сама** через ограждения — без подтверждения безопасности.
- Номера DO на шкафу могут отличаться от DO3 в шаблоне.
- OPC UA — к PLC ячейки (деталь на месте и т.п.), это отдельно от FTP на сам контроллер.
