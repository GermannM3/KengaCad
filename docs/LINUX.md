# Linux и AppImage

## Текущий продукт (KengaCAD Professional 2.x)

Приложение в папке `KengaCAD/` собрано на **WPF** и целится в **`net8.0-windows`**. Графический стек WPF официально работает только на Windows.

Поэтому:

- **AppImage из этой кодовой базы собрать нельзя** — в установочный образ нечего положить как нативное Linux-приложение.
- Скрипт-заглушка `installers/linux/build_appimage.sh` завершается с кодом 2 и напоминает об этом ограничении.

## Что можно сделать дальше

1. **Отдельный порт UI** на кроссплатформенный стек (.NET + Avalonia, MAUI и т.д.) с общей бизнес-логикой — тогда возможны `linux-x64` publish и упаковка в AppImage.
2. **Легаси PyQt** в `_legacy/` — теоретически можно собрать AppImage для Linux, но это другая архитектура, зависимости и линия версий; нужен отдельный CI и тесты.

## Windows-установщик

Полноценная автономная сборка: `dotnet publish ... --self-contained true -r win-x64`. Папка `publish/` занимает порядка **150–200 МБ** (среда .NET + WPF + SkiaSharp). Установщик Inno Setup сжимает это до **~50 МБ**; установка может занимать **несколько секунд** на быстром диске — это нормально: зависимости уже внутри архива, а не качаются из интернета.
